# long-ppt-hybrid

`long-ppt-hybrid` 负责复杂视觉页面的混合重建路线。

## 目标

在视觉保真、编辑效率和后续可维护性之间做最实用的平衡。

## 最适合的页面

- 仪表盘页
- 监控截图页
- 大屏页
- 界面型截图页
- 需要保留原图质感的复杂页面

## 核心原则

- 不强行全原生重建
- 图片密集区域尽量保留
- 真正要改的部分再做成可编辑对象
- 优先做高价值编辑区，不做低价值重画

## 典型可编辑区域

- 标题
- 长文本
- 标签
- 表格
- 模块说明
- 指标摘要

## 路线判断

优先选 `long-ppt-hybrid` 的情况：

- 页面像截图
- 页面像监控大屏
- 页面全重建成本太高
- 原始视觉保真比结构原生化更重要

## 与其他子 skill 的关系

- `long-ppt-core`：判断这页应走 hybrid
- `long-ppt-image`：在必要时先重做视觉底板
- `long-ppt-hybrid`：真正执行混合可编辑重建
- `long-ppt-native`：当页面结构化程度更高时可作为替代路线
