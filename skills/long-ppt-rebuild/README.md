# long-ppt-rebuild

`long-ppt-rebuild` 是 `Long-ppt skill` 里专门处理“现有视觉页 -> 可编辑 PPT”重建问题的一条路线。

它通常接的不是空白页，而是这些输入：

- 一页现成的 PPT 截图
- 一页 PNG / JPG / WebP 视觉稿
- 图片版 PPTX
- 需要尽量按原版式重建成可编辑 PPT 的整页内容

## 和其他 skill 的关系

这条路线和其他模块的边界比较清楚：

- `long-ppt-image`
  负责先生成新图或重画底板
- `long-ppt-hybrid`
  负责截图保真，同时把关键信息做成可编辑
- `long-ppt-native`
  负责结构图、附件页、规则明确页面的原生对象重建
- `long-ppt-rebuild`
  负责以“现有视觉页”为起点，按原版式尽量还原为可编辑 PPT

可以把它理解为：

`long-ppt-rebuild = 单页图片稿 / 截图页 / 整页视觉稿的可编辑 PPT 还原能力`

## 更适合它的场景

- 用户给的是一页现成图片，不是结构化文档
- 用户明确希望“这页变成可编辑 PPT”
- 页面既有文字，也有卡片、连线、标签、框体等视觉元素
- 原始文件已经是图片页，但后续还需要持续编辑

## 它的核心方法

不是盲目描图，而是：

1. 先拆视觉结构
2. 再判断哪些元素该用原生对象重建
3. 哪些元素保留为局部图片资产
4. 最后输出可编辑 PPTX

也就是说，它强调的是：

- 版式尽量贴近原稿
- 重要文字必须可编辑
- 结构对象尽量独立
- 结果可渲染验证

## 和 `ppt2` 的关系

这条路线沉淀的是“按原版式重建现有视觉页”的方法，已经被整理进 `Long-ppt` 体系。

它保留了几个关键原则：

- 先理解页面结构，不是直接整页贴图
- 尽量按对象级别重建，而不是只做截图覆盖
- 强调可编辑与后续维护能力
- 重建后需要渲染验证

## 一句话定位

`long-ppt-rebuild` 解决的是把图片页、截图页、整页视觉稿，尽量还原成“肉眼接近原图”的可编辑 PPT。
