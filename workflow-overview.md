# Long-ppt 总流程

`Long-ppt skill` 的整体工作流，可以理解成下面这条链路：

## 1. 输入现有材料

输入可以是：

- PPT
- Word
- 方案稿
- 演讲稿
- 项目材料
- 零散笔记

## 2. 先由 `long-ppt-core` 做理解和提炼

这一层负责：

- 内容总结
- 结构提炼
- 大纲重写
- 页面路线判断

## 3. 按页面类型分流

- 页面需要先加强视觉：`long-ppt-image`
- 页面起点是一页现成视觉稿 / 图片页：`long-ppt-rebuild`
- 页面是截图 / 大屏 / 仪表盘：`long-ppt-hybrid`
- 页面是架构图 / 附件页 / 框架图：`long-ppt-native`
- 页面依赖宏工作流：`long-ppt-vba`

## 4. 最终交付

目标不是“做完一页”，而是得到一套真正可交付的结果：

- 可展示
- 可编辑
- 可继续维护

## 一句话总结

Long-ppt 不只是“生成 PPT”，而是“基于现有材料，把 PPT 做成真正能交付的版本”。
